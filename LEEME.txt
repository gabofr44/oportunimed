CÓMO APLICAR ESTA ACTUALIZACIÓN (4 funcionalidades)
=====================================================

Contiene: compartir oportunidad, fecha de "última verificación",
nivel educativo múltiple, y recordatorio de deadlines por correo.

1) INSTALA LA LIBRERÍA NUEVA (antes de copiar los archivos)

   En PowerShell, parado en tu carpeta del proyecto:

     npm install resend

2) DESCOMPRIME este ZIP dentro de tu carpeta del proyecto (igual que la
   vez pasada) - dile que sí a reemplazar cuando pregunte.

3) Revisa con git status que se detectaron los cambios esperados:
   - Modificados: src/actions/admin.ts, src/actions/opportunities.ts,
     src/app/comparar/page.tsx, src/app/opportunities/[id]/page.tsx,
     src/app/opportunities/page.tsx, src/components/admin/OpportunityManager.tsx,
     src/types/index.ts
   - Nuevos: src/app/api/cron/deadline-reminders/route.ts,
     src/components/opportunities/ShareButton.tsx,
     3 migraciones nuevas (016, 017, 018), vercel.json

4) AGREGA 2 VARIABLES DE ENTORNO NUEVAS

   En tu .env.local Y en Vercel (Settings -> Environment Variables),
   igual que hiciste con SUPABASE_SERVICE_ROLE_KEY:

     RESEND_API_KEY=<la sacas de resend.com, ver abajo>
     CRON_SECRET=<cualquier texto largo y aleatorio que tu inventes>

   Para RESEND_API_KEY:
   a) Crea una cuenta gratis en https://resend.com (con tu correo o GitHub)
   b) En el dashboard, ve a "API Keys" -> "Create API Key" -> copia la clave
   c) Nota: el plan gratis te deja enviar correos de prueba solo a tu propio
      correo verificado, a menos que verifiques un dominio propio (ve a
      "Domains" en Resend y sigue sus instrucciones - necesitas acceso a
      la configuracion DNS de tu dominio). Mientras no verifiques un dominio,
      cambia esta linea en route.ts para pruebas:
        from: "Oportunimed <onboarding@resend.dev>"
      (resend.dev es su dominio de pruebas, siempre funciona)

   Para CRON_SECRET: es solo una contraseña que tú inventas, para que nadie
   más que Vercel pueda activar el envío de correos. Ejemplo:
   CRON_SECRET=x7k9mQp2vL8nR4wT6yB1

   En Vercel, cuando agregues CRON_SECRET como variable de entorno, Vercel
   Cron la usa automáticamente para autenticarse - no tienes que hacer nada
   más ahí.

5) Sube el código:

   git add .
   git commit -m "feat: compartir, fecha de verificacion, niveles multiples, recordatorio de correo"
   git push

6) Corre las 3 migraciones SQL nuevas en Supabase, EN ESTE ORDEN:
   - 016_add_last_verified.sql
   - 017_educational_level_array.sql   <- esta es la mas importante, revisa
     que corra sin error antes de seguir (convierte una columna de texto a
     lista)
   - 018_add_reminder_tracking.sql

7) Después del deploy en Vercel:
   - Ve a Vercel -> tu proyecto -> pestaña "Cron Jobs" (o "Settings" ->
     "Cron Jobs") y confirma que aparece el job "/api/cron/deadline-reminders"
     corriendo diario. Vercel también te deja darle "Run" manual desde ahí
     para probarlo sin esperar al horario programado.
   - Prueba el botón "Compartir" en el detalle de una convocatoria
   - En /admin -> pestaña Convocatorias, prueba el ícono de check verde para
     marcar una como verificada, y confirma que aparece la fecha
   - Guarda una oportunidad con deadline próximo en Favoritos, corre el cron
     manualmente desde Vercel, y revisa si te llega el correo (revisa spam)

NOTA sobre el nivel educativo múltiple: el formulario de admin para crear/
editar convocatorias todavía NO tiene un campo visual para elegir varios
niveles (antes tampoco tenía campo para elegir ni uno solo - ese dato
siempre se cargó por SQL directo). Si quieres poder editarlo desde el
panel admin, es una mejora aparte que podemos hacer después.
